## Task 4. STEP 3 — 본문 편집기 (Human Confirm)

기존 TipTap/Quill 편집기 기반으로 아래 추가:

### 4-1. 글자수 카운터

편집기 하단에 실시간 글자수 표시.  
- `< 1500자`: 주황 경고 "권장 분량 미달"
- `1500 ~ 2500자`: 녹색 "권장"
- `> 2500자`: 파랑 "충분"

### 4-2. 우측 SEO 체크리스트 패널

편집기 우측에 고정 패널 (모바일: 하단 접이식).  
실시간 체크 항목:

```
✅ 타겟 키워드 제목 포함
✅ 키워드 밀도 (1~3%)
⬜ 이미지 최소 3장
⬜ H2 소제목 최소 2개
✅ 2500자 이상
```

- 키워드 밀도: `(키워드 등장 횟수 / 전체 단어 수) * 100`
- H2 개수: 본문 HTML에서 `<h2>` 태그 count
- 체크 통과: 초록, 미통과: 빨강/주황

### 4-3. 자동 저장

`useEffect` + `setTimeout` 30초마다:  
`updateContent(id, { body: currentBody, publish_status: 'draft' })`  
우측 상단에 "저장됨 HH:MM" 표시.

### 4-4. 단락 재작성

텍스트 선택 시 컨텍스트 메뉴 또는 툴바 버튼 [🔄 단락 재작성]:

```
POST /api/portal/rewrite-paragraph
Body: { content_id, paragraph: selectedText, keyword, context: surroundingText }
Response: { rewritten: "..." }
```

`/api/portal/rewrite-paragraph` API 라우트 신규 생성:
- Claude Haiku 사용
- 시스템 프롬프트: "다음 단락을 SEO에 최적화된 한국어로 자연스럽게 재작성해라. 키워드: {keyword}"
- 스트리밍 없음, 단순 호출

## Task 6. 자동발행 설정 고도화

`portal-blog-unified-client.tsx` 의 자동발행 탭 확장.  
`auto_publish_settings` 테이블의 기존 JSONB에 신규 키 추가 (SELECT → spread → UPDATE 패턴).

### 6-1. 키워드 풀 설정

- 현재 활성 키워드 목록 드래그앤드롭 정렬 UI
- 선택된 키워드 ID 배열을 `settings JSONB → keyword_pool: string[]` 에 저장
- 발행 방식 라디오: "순서대로" / "순위 낮은 순"
- 중복 방지 토글 (30일 이내 재발행 방지): `settings → no_duplicate_days: 30`

```typescript
// 저장 패턴 (JSONB spread)
const current = await getAutoPublishSettings(clientId)
await updateAutoPublishSettings(clientId, {
  ...current,
  settings: {
    ...current.settings,
    keyword_pool: selectedKeywordIds,
    order_by: 'sequence' | 'rank_asc',
    no_duplicate_days: 30
  }
})
```

### 6-2. 발행 주기 설정

- 라디오: 매일 / 주 N회 / 월 N회
- 주 N회 선택 시 요일 체크박스 (월~일)
- 월 N회 선택 시 날짜 선택 (1~28일)
- 발행 시간 피커 (HH:MM)
- 저장 → `settings → schedule: { type, days, time }` JSONB 저장

### 6-3. 처음 3건 컨펌 안내

ON 전환 시 모달:  
"자동발행을 켭니다. 처음 3건은 발행 전 알림을 받고 직접 확인 후 발행됩니다."  
`settings → confirm_count: 0` (발행 시마다 카운트 증가, 3건 이후 자동발행)
