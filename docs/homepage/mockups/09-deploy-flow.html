☁️ Vercel 배포 플로우

어드민에서 버튼 클릭 → Vercel API로 자동 배포 → 서브도메인 활성화까지의 전체 파이프라인

어드민
빌드 & 배포 클릭
👤

1. 배포 요청

어드민 대시보드에서 프로젝트의 "빌드 & 배포" 버튼을 클릭합니다.
템플릿 선택, 환경변수 확인 후 배포를 시작합니다.

POST /api/homepage/deploy
// Body
{ "projectId": "uuid", "templateId": "modern-minimal" }
Server Action
deployHomepage()
⚙️

2. Vercel 프로젝트 생성 & 환경변수 주입

Vercel REST API로 프로젝트를 생성하고, 필요한 환경변수를 자동 주입합니다.

// Vercel API Calls:
POST api.vercel.com/v10/projects
POST api.vercel.com/v10/projects/:id/env
POST api.vercel.com/v13/deployments
NEXT_PUBLIC_SITE_URL
NEXT_PUBLIC_PROJECT_ID
SUPABASE_URL
SUPABASE_ANON_KEY
SUPABASE_SERVICE_KEY
NEXT_PUBLIC_KAKAO_ID
Vercel
빌드 진행
▲

3. 빌드 & 배포 실행

Vercel에서 Next.js 빌드를 실행합니다. 빌드 중 getHomepageConfig()로 Supabase 데이터를 fetch하여 정적 페이지를 생성합니다.

빌드 진행 중... 45s
✅ Installing dependencies... done
✅ Building Next.js app...
⏳ Generating static pages (4/7)...
⏳ Deploying to edge network...
Webhook
빌드 완료 알림
🔔

4. 빌드 완료 → 상태 업데이트

Vercel 웹훅이 빌드 완료를 알리면, homepage_projects.status를 'preview'로 자동 변경합니다.

POST /api/homepage/deploy/webhook
// Vercel sends deployment.ready event
// → UPDATE homepage_projects SET status='preview'
DNS
서브도메인 연결
🌐

5. 서브도메인 자동 설정

Vercel 도메인 API로 {업체명}.waide.kr 서브도메인을 자동 연결합니다.

POST api.vercel.com/v10/projects/:id/domains
{ "name": "gangnam-int.waide.kr" }
어드민
라이브 전환
✅

6. 라이브 전환 완료

프리뷰 확인 후 "라이브 전환" 버튼을 클릭하면 status='live'로 변경됩니다.
고객이 gangnam-int.waide.kr에서 홈페이지를 볼 수 있습니다.

POST /api/homepage/deploy/promote
// → UPDATE homepage_projects SET status='live'
// → 커스텀 도메인 설정 (선택)
