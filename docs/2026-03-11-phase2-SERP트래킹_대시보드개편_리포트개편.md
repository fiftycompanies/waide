### 1-1. `getPortalHealthScore(clientId: string)`

마케팅 건강 점수 4개 세부 항목 계산 후 반환.

```typescript
// 반환 타입
{
  totalScore: number,        // 가중 평균 (0~100)
  grade: 'A'|'B'|'C'|'D'|'F',
  blog: {
    score: number,
    // 어드민 /api/cron/grading 의 블로그 점수 로직 그대로 사용
    // account_grades 테이블에서 해당 client의 blog 점수 조회
    // 없으면 contents 테이블에서 최근 30일 published count × 가중치로 계산
  },
  seo: {
    score: number,
    // brand_analyses.seo_audit JSONB 파싱
    // seo_audit 내 score 필드 사용 (기존 어드민 SEO 점수 로직과 동일)
  },
  serp: {
    score: number,
    // keyword_visibility 테이블에서 clientId의 활성 키워드 점유율
    // SELECT count(*) FILTER (WHERE rank <= 30) / count(*) * 100
    // getClientRankings(clientId) 함수 활용
  },
  place: {
    score: number,
    // keywords 테이블에서 clientId의 메인 키워드 3개 (priority 순)
    // current_rank_naver_pc 기준으로 점수 환산:
    //   1~3위: 100점, 4~10위: 80점, 11~20위: 60점, 21~30위: 40점, 31위↓: 20점, null: 0점
    // 3개 평균
  },
  prevMonthScore: number | null,  // 전월 brand_analyses.marketing_score
  delta: number | null,
}
```

### 1-2. `getPortalUrgentBannerCondition(clientId: string)`

긴급 후킹 배너 표시 조건 계산.

```typescript
// 반환 타입
{
  show: boolean,
  reason: 'rank_drop' | 'no_publish_14d' | 'no_publish_this_month' | 'low_quota',
  criticalKeywordId: string | null,  // 가장 위험한 키워드 ID
  message: string,
}
// 조건 (OR):
// A: keyword_visibility에서 rank > 20 OR 7일 연속 하락 키워드 존재
// B: contents.published_at max < 14일 전
// C: 이번달 published count = 0
// D: client_points.balance <= 3 (잔여 포인트 3 이하)
```

### 1-3. `getPortalSerpPage(clientId: string)`

SERP 트래킹 페이지 전용 데이터.

```typescript
// 반환 타입
{
  keywords: Array<{
    id: string,
    keyword: string,
    platform: 'naver' | 'google',
    currentRank: number | null,
    rankChange: number | null,
    sparkline: number[],       // 최근 7일 순위 배열 (null은 0으로)
    publishedUrl: string | null,
    statusBadge: 'top' | 'mid' | 'danger' | 'invisible',
    lastPublishedAt: string | null,
  }>,
  missingKeywords: string[],   // rank IS NULL OR rank >= 50
  criticalKeywordId: string | null,
}
// statusBadge 기준: rank <= 10: top, 11~30: mid, 31~50: danger, 50+|null: invisible
// sparkline: getSerpByKeyword() 최근 7일 데이터 활용
// lastPublishedAt: contents 테이블에서 keyword_id 기준 최신 published_at
```

## Task 2. SERP 트래킹 신규 페이지 (`/portal/serp`)

`app/(portal)/portal/serp/page.tsx` 신규 생성 (server component).  
`components/portal/portal-serp-client.tsx` 신규 생성.

### 컴포넌트 구성

**상단 액션바:**
- 플랫폼 탭: 전체 / 네이버 / 구글 (쿼리파라미터 `?platform=`)
- 상태 필터 드롭다운: 전체 / 상위 / 위험 / 미노출 (다중 선택)
- [+ 키워드 추가] 버튼 → 모달

**키워드 추가 모달:**
- 키워드 입력 텍스트
- 플랫폼 라디오 (네이버/구글)
- 입력 시 `queryKeywordVolume()` 자동 조회 → 검색량 표시
- [저장] → `createKeyword({ keyword, client_id: clientId, status: 'active', is_tracking: true })`

**누락 키워드 감지 배너:**
- `missingKeywords.length > 0` 일 때 테이블 상단에 표시
- [지금 발행하기] → `/portal/blog/write?keyword_id={criticalKeywordId}&from=serp_alert`

**키워드 순위 테이블:**
- 컬럼: 키워드 + 플랫폼 아이콘 / 현재 순위 / 순위 변화 / 7일 sparkline / 연결 URL / 상태 뱃지 / 최근 발행일 / 액션
- Sparkline: SVG 인라인 라인차트 (Recharts `<Sparklines>` 또는 간단 SVG)
- 행 클릭 → 해당 행 하단에 30일 추이 차트 슬라이드다운 (인라인 expand)
- 발행 버튼 → `/portal/blog/write?keyword_id={id}`
- 삭제 → 확인 다이얼로그 → `archiveKeyword(id)`

**30일 추이 차트 (슬라이드다운):**
- `getSerpByKeyword(keywordId)` 데이터 사용
- Recharts LineChart, Y축 역방향 (domain 역전: [50, 1])
- X축: 날짜 MM/DD
- 발행 마커: `contents` 테이블에서 해당 `keyword_id`의 `published_at` 날짜에 📝 아이콘 표시 (ReferenceLine)

## Task 3. 대시보드 마케팅 건강 점수 카드 개편

`app/(portal)/portal/page.tsx` 수정.  
기존 마케팅 점수 원형 차트 섹션을 아래로 교체 (기존 코드 주석 처리 후 새 섹션 추가).

**건강 점수 카드 UI:**

```
┌─────────────────────────────────────┐
│  💯 마케팅 건강 점수          [A] 82  │
│  ████████████░░░░  전월 대비 +5↑     │
│                                     │
│  블로그 85  SEO 78  SERP 90  플레이스 75 │
│  [점수 상세 보기]  [점수 올리기 →]   │
└─────────────────────────────────────┘
```

- `getPortalHealthScore(clientId)` 서버에서 호출 후 props로 전달
- 등급 뱃지: A=초록, B=파랑, C=노랑, D=주황, F=빨강
- "점수 올리기" 버튼: 4개 항목 중 최저 점수 항목에 따라 동적 라우팅
  - 블로그 최저 → `/portal/blog/write`
  - SEO 최저 → `/portal/analysis`
  - SERP 최저 → `/portal/blog/write`
  - 플레이스 최저 → `/portal/analysis`

**발행 현황 카드 (기존 카드 수정):**
- "잔여 횟수" → "잔여 포인트"로 텍스트 변경
- 데이터: `getPortalPointBalance()` → `balance` 값 표시
- 잔여 포인트 0 이하 시 [플랜 업그레이드] 버튼 표시 → `/portal/settings?tab=subscription`

**긴급 후킹 배너:**
- `getPortalUrgentBannerCondition(clientId)` 서버에서 호출
- `show: true` 일 때 대시보드 최상단에 빨간 배너 표시
- [지금 발행하기] → `/portal/blog/write?keyword_id={criticalKeywordId}&urgent=true`
- [×] 닫기 → `sessionStorage`에 오늘 날짜 저장, 당일 재노출 없음

**키워드 순위 요약 카드 (기존 확장):**
- 기존 TOP3/TOP10 표시 유지
- 각 키워드 행에 7일 mini sparkline 추가 (SVG 간단 구현)
- delta ≥ 10 하락인 키워드 행에 [지금 발행] 버튼 표시 → `/portal/blog/write?keyword_id={id}`
