## Task 1. DB 마이그레이션 (가장 먼저 실행)

`apps/web/supabase/migrations/` 에 새 마이그레이션 파일 생성.

```sql
-- contents.publish_status에 scheduled, pending_confirm 상태 추가
ALTER TABLE contents DROP CONSTRAINT IF EXISTS contents_publish_status_check;
ALTER TABLE contents ADD CONSTRAINT contents_publish_status_check 
  CHECK (publish_status IN ('draft','review','approved','published','rejected','archived','scheduled','pending_confirm'));

-- 예약 발행 시간 컬럼 추가
ALTER TABLE contents ADD COLUMN IF NOT EXISTS scheduled_at TIMESTAMPTZ;
```

마이그레이션 적용 후 빌드 오류 없는지 확인.

## Task 3. 블로그 관리 통합 (`/portal/blog`)

### 3-1. 탭 구조 추가

`app/(portal)/portal/blog/page.tsx` 를 server component로 교체.  
`components/portal/portal-blog-unified-client.tsx` 신규 생성.

**4탭 구조:**

```
탭 1: 블로그 목록 (기본)
탭 2: AI 작성 → /portal/blog/write 로 이동 (탭 클릭 시 라우팅)
탭 3: 자동발행 설정 (기존 portal-blog-client.tsx 내 자동발행 섹션 이동)
탭 4: 콘텐츠 관리 (기존 portal-contents-client.tsx 그대로 이동)
```

### 3-2. 블로그 목록 탭 (탭 1)

기존 `portal-blog-client.tsx`의 "발행 이력" 섹션을 확장.  
**3개 서브탭: 발행완료 / 예약 / 초안**

각 서브탭 데이터:
- 발행완료: `getPortalContentsV2()` → `publish_status = 'published'` 필터
- 예약: `getPortalContentsV2()` → `publish_status = 'scheduled'` 필터  
- 초안: `getPortalContentsV2()` → `publish_status = 'draft'` 필터

각 행에 표시: 제목, 연결 키워드 칩, 플랫폼 아이콘, 발행일, SERP 순위 뱃지(발행완료만)

행 버튼:
- 발행완료: [상세 보기] → `/portal/blog/[id]` / [재발행] (확인 다이얼로그 후 `executePublish()` 재호출)
- 예약: [예약 취소] → `updateContent(id, {publish_status:'draft', scheduled_at: null})`
- 초안: [이어쓰기] → `/portal/blog/write?post_id={id}&mode=edit` / [삭제] → `deleteContent(id)`

### 3-3. 콘텐츠 관리 탭 (탭 4)

기존 `portal-contents-client.tsx` 컴포넌트를 그대로 import해서 렌더링.  
별도 수정 없이 재사용.

### 3-4. 자동발행 탭 (탭 3)

기존 `portal-blog-client.tsx`의 자동발행 관련 섹션(ON/OFF 토글, 채널 설정, 이력 테이블)을 그대로 재사용.

- [숨기기] → `updateContent(id, {is_active: false})` → 목록 복귀
