## Task 1. DB 마이그레이션

`apps/web/supabase/migrations/` 에 신규 파일 생성.

```sql
-- 알림 테이블
CREATE TABLE IF NOT EXISTS notifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  client_id UUID NOT NULL REFERENCES clients(id) ON DELETE CASCADE,
  type VARCHAR(50) NOT NULL CHECK (type IN (
    'rank_drop', 'rank_rise', 'publish_complete', 'quota_warning', 'auto_publish_confirm'
  )),
  title TEXT NOT NULL,
  body TEXT,
  metadata JSONB DEFAULT '{}',
  -- metadata 예시: { keyword_id, content_id, old_rank, new_rank, platform }
  is_read BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_notifications_client_id ON notifications(client_id);
CREATE INDEX IF NOT EXISTS idx_notifications_is_read ON notifications(is_read);
CREATE INDEX IF NOT EXISTS idx_notifications_created_at ON notifications(created_at DESC);

-- 알림 설정 테이블
CREATE TABLE IF NOT EXISTS notification_settings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  client_id UUID NOT NULL UNIQUE REFERENCES clients(id) ON DELETE CASCADE,
  settings JSONB DEFAULT '{
    "rank_drop": true,
    "rank_rise": true,
    "publish_complete": true,
    "quota_warning": true,
    "email_enabled": false,
    "push_enabled": false
  }',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

## Task 2. 알림 Server Actions (`lib/actions/notification-actions.ts`)

신규 파일 생성. `createAdminClient()` 사용.

```typescript
// 알림 목록 조회
export async function getNotifications(clientId: string, limit = 50)
// → notifications WHERE client_id = clientId ORDER BY created_at DESC LIMIT limit

// 읽지 않은 알림 수
export async function getUnreadCount(clientId: string)
// → COUNT WHERE is_read = false

// 모두 읽음 처리
export async function markAllRead(clientId: string)
// → UPDATE SET is_read = true WHERE client_id = clientId

// 단일 읽음 처리
export async function markRead(notificationId: string)

// 알림 생성 (내부 함수, export)
export async function createNotification(params: {
  clientId: string,
  type: string,
  title: string,
  body?: string,
  metadata?: Record<string, any>
})

// 알림 설정 조회
export async function getNotificationSettings(clientId: string)
// → notification_settings WHERE client_id = clientId
// 없으면 default 값 반환

// 알림 설정 저장
export async function updateNotificationSettings(clientId: string, settings: Record<string, any>)
// SELECT → spread → UPDATE 패턴 (JSONB)
```

## Task 4. 알림 센터 페이지 (`/portal/notifications`)

Phase 1에서 placeholder로 만든 페이지를 완성.  
`app/(portal)/portal/notifications/page.tsx` 수정.  
`components/portal/portal-notifications-client.tsx` 신규 생성.

### 알림 피드

- `getNotifications(clientId)` 데이터 표시
- 타입별 아이콘: rank_drop=🔴, rank_rise=🟢, publish_complete=📝, quota_warning=💳, auto_publish_confirm=⚡
- 상대 시간 표시 (3분 전, 2시간 전)
- [모두 읽음] 버튼 → `markAllRead(clientId)` → 리렌더
- 읽지 않은 알림 배경색 구분 (연한 파랑)

**알림별 CTA 버튼 (타입별 동적):**

| 타입 | 버튼 | 이동 경로 |
|------|------|----------|
| rank_drop | 지금 발행 | `/portal/blog/write?keyword_id={keyword_id}` |
| rank_rise | 확인하기 | `/portal/serp` |
| publish_complete | 글 보기 | `/portal/blog/{content_id}` |
| quota_warning | 포인트 충전 | `/portal/settings?tab=subscription` |
| auto_publish_confirm | 확인 후 발행 | `/portal/blog/write?post_id={content_id}&mode=confirm` |

### 알림 설정 섹션

- `getNotificationSettings(clientId)` 조회
- 알림 종류별 토글 스위치 (rank_drop/rank_rise/publish_complete/quota_warning)
- 이메일 알림 토글 (이메일 주소는 users 테이블에서 자동 조회)
- [저장] → `updateNotificationSettings(clientId, settings)` → toast "저장됨"

### 네비 뱃지

`portal-shell.tsx` 의 알림 센터 메뉴 아이템에 읽지 않은 알림 수 뱃지 표시:  
`getUnreadCount(clientId)` 서버에서 조회 → 숫자 뱃지.
